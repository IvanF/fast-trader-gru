## Ответ на вопрос: Да, это возможно

Текущая архитектура уже собирает все данные, необходимые для обучения такой нейросети. Вот что уже есть:

| Данные | Источник | Где хранятся |

### Текущая проблема

Сейчас optimal SL/TP вычисляется **после** сделки как `MAE × 1.1` и `MFE × 0.9` — это retrospective, не predictive. Нейросеть должна **предсказывать** оптимальные расстояния **до** входа в сделку.

## Архитектура: ExitOptimizer Network

### Входные фичи (features)

```
Input dim = 128 (state_vector) + 8 (memory) + 12 (extras) = 148

Extras:
- direction (one-hot: LONG=1, SHORT=0)
- regime (one-hot: Choppy, Trending, Breakout)
- confidence (float)
- vol_mult (float)
- obi (float, order book imbalance)
- trend_5m, trend_15m (float)
- funding_rate (float)
- avg_sl_distance_pct (float, среднее из FAISS neighbors)
- avg_tp_distance_pct (float, среднее из FAISS neighbors)
- salvageable_ratio (float, доля salvageable neighbors)
```

### Выходные фичи (labels)

```
Output dim = 3:
- optimal_sl_pct (float, предсказанное расстояние до SL)
- optimal_tp_pct (float, предсказанное расстояние до TP)
- trade_score (float, 0-1, вероятность прибыльной сделки)
```

### Модель

```python
class ExitOptimizer(nn.Module):
    def __init__(self, in_dim=148, hidden=64):
        self.net = nn.Sequential(
            nn.Linear(in_dim, hidden),
            nn.ReLU(),
            nn.Dropout(0.3),
            nn.Linear(hidden, 32),
            nn.ReLU(),
            nn.Linear(32, 3),
            nn.Sigmoid()  # все выходы [0,1], масштабируются отдельно
        )
    
    def forward(self, x):
        out = self.net(x)
        sl_pct = out[:, 0] * 0.03      # [0, 3%]
        tp_pct = out[:, 1] * 0.05      # [0, 5%]
        score = out[:, 2]              # [0, 1]
        return sl_pct, tp_pct, score
```

### Labels (целевые значения)

Из历史 сделок:
- `optimal_sl_pct` = `abs(MAE) × 1.1` (если MAE есть)
- `optimal_tp_pct` = `abs(MFE) × 0.9` (если MFE есть)
- `trade_score` = 1.0 если PnL > 0, иначе 0.0

### Потеря (loss)

```python
loss = MSE(sl_pred, sl_true) + MSE(tp_pred, tp_true) + BCE(score_pred, score_true)
```

### Инференс (как используется)

При генерации сигнала:
1. FusionModel → state_vector (128-dim)
2. Memory query → v_memory (8-dim) + dynamic_sl/tp из neighbors
3. **ExitOptimizer(state_vector + extras) → predicted_sl_pct, predicted_tp_pct, trade_score**
4. exit_plan.py использует predicted_sl/tp вместо фиксированных min/max
5. Если trade_score < порога → сигнал отклоняется

### Порядок реализации

1. **ExitOptimizer модель** (nn_models.py) — определение архитектуры
2. **Training pipeline** (train.py) — загрузка历史 данных из InfluxDB + FAISS,训练 на MAE/MFE labels
3. **Inference integration** (inference.py + engine.py) — вызов ExitOptimizer при сигнале
4. **Exit plan integration** (exit_plan.py) — использование predicted sl/tp
5. **Валидация** — сравнение PnL до/после внедрения

### Ожидаемый эффект

- **TP**: будет ставиться на оптимальное расстояние на основе исторических MFE паттернов
- **SL**: будет адаптироваться к реальной волатильности на основе MAE паттернов
- **R:R**: должен вырасти с текущих 0.41:1 до 1.0+:1
- **Win rate**: может снизиться (более избирательные входы), но профит вырастет

### Риски

- Нужно ≥1000 сделок с MAE/MFE для обучение (сейчас ~100)
- Модель может overfit на исторические паттерны
- Инференс добавляет ~1-2ms задержки (принимаемо)
