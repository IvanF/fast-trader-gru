# Верификация исправлений: Confidence Threshold + EV Gate matches

## Что изменено
1. `.env` — `CONFIDENCE_THRESHOLD` 0.25→0.15, `LONG_CONFIDENCE_THRESHOLD` 0.25→0.15
2. `ml_engine/src/memory.py:149` — `"matches": int(k)` → `"matches": long_matches + short_matches`
3. `ml_engine/src/memory.py:89-90` — добавлены недостающие поля в ветку пустого индекса

## План верификации

### Шаг 1: Unit-тест memory.py
Создать `ml_engine/src/memory_test.py` — проверяет:
- `query()` с пустым индексом возвращает `matches=0`, `long_matches=0`, `short_matches=0`
- `query()` с записями LONG/SHORT возвращает `matches = long_matches + short_matches`
- `matches` ≠ k (раньше был равен k=10)
- Граничные случаи: все HOLD, все LONG, все SHORT

### Шаг 2: Unit-тест confidence threshold
Создать `ml_engine/src/config_test.py` — проверяет:
- `Config.load()` с `CONFIDENCE_THRESHOLD=0.15` → `cfg.confidence_threshold == 0.15`
- `Config.load()` с `LONG_CONFIDENCE_THRESHOLD=0.15` → `cfg.long_confidence_threshold == 0.15`

### Шаг 3: Запуск тестов
```bash
cd ml_engine && python -m pytest src/memory_test.py src/config_test.py -v
# Или без pytest:
cd ml_engine && python src/memory_test.py && python src/config_test.py
```

### Шаг 4: Проверка в контейнере (опционально)
```bash
docker compose restart ml_engine
docker compose logs -f ml_engine --tail=50 | grep -E "EV gate|PATTERN MEMORY|hold_low_conf"
```
Ожидаемое поведение:
- Меньше `hold_low_conf` (confidence 0.15-0.25 теперь проходят)
- EV Gate: `matches` теперь реальное число, фильтр работает точнее
