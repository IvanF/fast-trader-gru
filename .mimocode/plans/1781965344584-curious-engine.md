# Plan: Повышение confidence до 0.80 для убыточных символов в похожих сетапах

## Проблема

VANRYUSDT: 19 losses, HMSTRUSDT: 17 losses, 1000PEPEUSDT: 13 losses за 8 часов. Система входит в один и тот же убыточный символ снова и снова. Flip не работает (30% WR). Существующие фильтры слишком мягкие — confidence cap 0.75 позволяет входить.

**Директива пользователя**: не блокировать, а **повысить уверенность до 0.80** для тех монет, которые принесли убыток в похожем сетапе.

## Решение: Symbol+Setup Level Confidence Escalation

### 1. Symbol+Setup tracker в ML engine

**Файл:** `ml_engine/src/engine.py`

Добавить трекер убытков по **symbol+regime+direction**:

```python
# В __init__:
self._symbol_setup_losses: dict[str, list[float]] = {}  # "VANRYUSDT_Short_Choppy" -> [pnl1, pnl2, ...]

# В _listen_execution_results, при pnl < 0:
key = f"{symbol}_{direction}_{regime}"
self._symbol_setup_losses.setdefault(key, []).append(pnl)
if len(self._symbol_setup_losses[key]) > 50:
    self._symbol_setup_losses[key] = self._symbol_setup_losses[key][-50:]
```

### 2. Symbol+Setup confidence escalation

**Файл:** `ml_engine/src/engine.py`

После динамического confidence (Filter 9), добавить escalation по symbol+setup:

```python
# Symbol+Setup loss escalation
setup_key = f"{symbol}_{direction}_{regime.value}"
setup_losses = self._symbol_setup_losses.get(setup_key, [])
if len(setup_losses) >= 3:
    avg_setup_loss = sum(setup_losses) / len(setup_losses)
    if avg_setup_loss < -0.10:
        # 3+ losses in same symbol+regime+direction with avg loss > $0.10
        escalation = 0.80  # raise threshold to 0.80
        base_threshold = max(base_threshold, escalation)
        logger.info("Symbol+Setup escalation: %s %s %s avg_loss=%.4f → threshold=%.3f",
                    symbol, direction, regime.value, avg_setup_loss, base_threshold)
```

**Эффект**: VANRYUSDT SHORT Choppy с 19 losses → avg_loss < -$0.10 → threshold поднимается до 0.80. Модель должна быть очень уверена, чтобы войти снова.

### 4. Pattern Memory: добавить symbol-level проверку

**Файл:** `ml_engine/src/online_learner.py` (should_avoid)

Pattern Memory должна учитывать **symbol** при проверке:

```python
def should_avoid(self, v_state, regime, direction, symbol=""):
    similar = self.find_similar(v_state, regime, direction)
    
    # Symbol-level: 3+ losses of same symbol → raise bar
    if symbol:
        symbol_losses = [p for p in similar if p.symbol == symbol and p.pnl < 0]
        if len(symbol_losses) >= 3:
            return True, np.mean([p.pnl for p in symbol_losses])
    
    # State-vector level (без изменений)
    ...
```

Вызов в engine.py:
```python

```
