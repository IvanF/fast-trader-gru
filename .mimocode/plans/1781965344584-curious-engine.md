# Plan: Multi-Task Learning — Trap Head для борьбы с ловушками

## Проблема
AsymmetricTradingLoss (2.5x penalty) заставляет модель массово предсказывать HOLD → val_acc < 0.45 → модель отвергается. Система топчется на месте.

## Решение: Trap Head (6-й логит)
Модель учится предсказывать направление **БЕЗ штрафов за убытки** + отдельно предсказывает «ловушку» (is_trap). Нет штрафов → модель не боится предсказывать направление → val_acc растёт.

## Изменения по файлам

### 1. `src/models/nn_models.py` — Архитектура

**DecisionMLP**: `Linear(32→6)` вместо `Linear(32→5)`
- logits[0:3] = LONG/SHORT/HOLD
- logits[3] = confidence
- logits[4] = vol_mult
- **logits[5] = is_trap** (новый)

**FusionModel**: без изменений (MACRO_DIM=17 на шаге 2).

### 2. `src/features.py` + `src/influx_join.py` — 3 новые фичи (MACRO_DIM 14→17)

| # | Фича | Описание |
|---|------|----------|
| 14 | `obi_reversal` | `max(obi_seq) - min(obi_seq)` — разворот баланса |
| 15 | `pre_entry_sweep` | 1 если был агрессивный тейкер против направления за 2s до entry |
| 16 | `fill_delay_norm` | Нормализованное время fill (0-1), долгий fill = нестабильность |

**features.py**: `feature_vector()` возвращает `(17,)` вместо `(14,)`.
**influx_join.py**: `_rows_to_sequences()` вычисляет 3 новых фичи из доступных данных.
**FusionModel**: `Linear(17→64)` вместо `Linear(14→64)`.

### 3. `scripts/train.py` — Новая Loss + промоция

**УДАЛИТЬ**: `AsymmetricTradingLoss` и `loss_penalty_weight`

**НОВАЯ** `DecoupledTrapLoss`:
```python
class DecoupledTrapLoss(nn.Module):
    def forward(self, logits, targets):
        dir_loss = F.cross_entropy(logits[:, :3], targets['direction'], label_smoothing=0.1)
        conf_loss = F.mse_loss(torch.sigmoid(logits[:, 3]), targets['confidence'])
        trap_loss = F.binary_cross_entropy_with_logits(logits[:, 5], targets['is_trap_label'])
        return dir_loss + 0.5 * conf_loss + 1.5 * trap_loss
```

**НОВАЯ промоция**:
```python
# val_dir_acc >= 0.38 (минимальное качество)
# val_trap_acc > prev_trap_acc + 0.03 (модель улучшилась в ловушках)
promote = (val_dir_acc >= 0.38) and (val_trap_acc > prev_trap_acc + 0.03)
```

### 4. `src/engine.py` — Trap Head в инференсе

В `_run_tick_prediction()` после `inference.decide()`:
```python
trap_prob = sigmoid(logits[5])
if trap_prob > 0.60:
    logger.info("Trap Head blocked signal for %s: trap_prob=%.3f", symbol, trap_prob)
    return None
```

### 5. `scripts/train.py` — Обогащение данных

В `build_joined_dataset()` вычислять 3 новые фичи из доступных данных InfluxDB:

| Фича | Источник | Алгоритм |
|------|----------|----------|
| `obi_reversal` | market_features OBI | max(obi_seq) - min(obi_seq) за 60 тиков |
| `pre_entry_sweep` | market_features trades | 1 если max(abs(signed_size)) > 3σ за 2s до entry |
| `fill_delay_norm` | trade_outcomes.holding_time_ms | min(holding / 60000, 1.0) |

### 6. Экспорт ONNX

Новый MLP экспортируется с `out_dim=6`. Go-парсер не затронут (получает готовый JSON из Redis).
