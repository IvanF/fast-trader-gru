# Экспорт и обучение ONNX моделей

> **For agentic workers:** REQUIRED SUB-SKILL: Use compose:subagent (recommended) or compose:execute to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Создать и обучить ONNX модели для ML engine, чтобы система начала генерировать торговые сигналы вместо HOLD.

**Architecture:** Bootstrap экспортирует случайно инициализированные модели → система собирает данные → train.py обучает модели на собранных данных → hot-swap заменяет модели.

**Tech Stack:** PyTorch, ONNX Runtime, InfluxDB, Redis

---

### Task 1: Экспорт bootstrap моделей

**Covers:** Создание начальных ONNX моделей для запуска системы

**Files:**
- Create: `ml_engine/models/orderbook_cnn.onnx`
- Create: `ml_engine/models/flow_gru_attention.onnx`
- Create: `ml_engine/models/decision_mlp.onnx`
- Create: `ml_engine/models/manifest.json`

- [ ] **Step 1: Установить зависимости**

```bash
cd ml_engine
pip install torch onnx onnxruntime
```

- [ ] **Step 2: Экспорт моделей**

```bash
cd ml_engine
python scripts/export_models.py
```

Expected: `"exported ONNX models to .../models manifest=bootstrap"`

- [ ] **Step 3: Проверить файлы**

```bash
ls -la ml_engine/models/
```

Expected: 4 файла — 3 .onnx + manifest.json

- [ ] **Step 4: Запустить систему**

```bash
docker compose up --build -d
```

- [ ] **Step 5: Проверить логи ML engine**

```bash
docker logs ftg-ml-engine --tail 20
```

Expected: логи `"loaded orderbook_cnn from ... providers=..."` для каждой модели

### Task 3: Обучение моделей на собранных данных

**Covers:** Замена bootstrap моделей обученными

**Предусловие:** Система работает ≥ 2 часов, в InfluxDB есть ≥ 5 сделок

- [ ] **Step 1: Проверить наличие данных**

```bash
docker compose exec ml_engine python -c "
from src.influx_store import InfluxStore
import os
s = InfluxStore(os.getenv('INFLUX_URL'), os.getenv('INFLUX_TOKEN'), os.getenv('INFLUX_ORG'), 'market_raw', 'market_features')
rows = s.query_trade_outcomes('-6h')
print(f'Trade outcomes: {len(rows)}')
"
```

Expected: ≥ 5

- [ ] **Step 2: Запустить обучение**

```bash
docker compose exec ml_engine python scripts/train.py \
  --symbols BTCUSDT,ETHUSDT \
  --hours 6 \
  --epochs 15 \
  --device cuda
```

Expected: `"training complete in ...s loss_delta=..."` + `"promoted version=..."`

- [ ] **Step 3: Проверить hot-swap**

```bash
docker logs ftg-ml-engine --tail 10 | grep "hot-swapped"
```

Expected: `"hot-swapped ONNX models -> version=..."`

- [ ] **Step 4: Проверить что модель загружена**

```bash
docker logs ftg-ml-engine --tail 5 | grep "loaded"
```

Expected: новые version в логах загрузки
