# Multi-Task Learning: Trap Head Implementation Plan

## Problem Summary
Current system has ~50% losing trades. `AsymmetricTradingLoss` (2.5x penalty on losses) causes DecisionMLP to mass-predict HOLD → val_acc drops below 0.45 → model rejected by promotion gate. System is stuck.

## Solution
Add a "Trap Head" (logit[5]) to DecisionMLP that predicts whether a trade is a trap (false breakout / stop hunt). Replace the punitive loss with a decoupled multi-task loss.

## Step 3: Decoupled Loss — `ml_engine/scripts/train.py`

### 3a. Delete `AsymmetricTradingLoss` class entirely

### 3b. Create `DecoupledTrapLoss`
```python
class DecoupledTrapLoss(nn.Module):
    def forward(self, logits, targets):
        # 1. Direction: clean cross-entropy, NO loss penalty
        dir_loss = F.cross_entropy(logits[:, :3], targets['direction'], label_smoothing=0.1)
        # 2. Confidence: MSE
        conf_loss = F.mse_loss(torch.sigmoid(logits[:, 3]), targets['confidence'])
        # 3. Trap head: BCE — the core learning signal from losing trades
        trap_loss = F.binary_cross_entropy_with_logits(logits[:, 5], targets['is_trap'])
        return dir_loss + 0.5 * conf_loss + 1.5 * trap_loss
```

### 3c. Update `JoinedDataset` to include `is_trap`
- Add `self.is_trap` array from data["is_trap"]
- Return `torch.tensor(self.is_trap[idx])` in `__getitem__`

### 3d. Update `train_epoch` and `eval_epoch`
- Replace `asym_loss(logits, direction, pnl)` with `DecoupledTrapLoss` using dict of targets
- Remove class balancing logic from old loss (keep stratified split)

### 3e. Remove `loss_penalty` from config/args
- Remove `--loss-penalty` arg from argparse
- Remove `loss_penalty_weight` from `Config` dataclass

## Step 4: Promotion Logic — `ml_engine/scripts/train.py`

### Replace entire validation/promotion block:
```python
# OLD: should_promote = (acc_improved or loss_improved) and acc_above_min and signal_rate_ok
# NEW:
val_trap_acc = <compute trap prediction accuracy on val set>
should_promote = (val_dir_acc > 0.38) and (val_trap_acc > prev_trap_acc + 0.03)
```

Key changes:
- Remove `MIN_VAL_ACC = 0.45` gate
- Add `val_trap_acc` computation: compare `round(sigmoid(logits[:,5]))` vs `is_trap` labels
- New gate: direction accuracy stays above 38% AND trap accuracy improved by >3%
- This lets the model freely predict HOLD for traps without being penalized
